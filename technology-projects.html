Nile Dot Com

Nile Dot Com was a Java event-driven GUI application built as an enterprise simulation of an online store. The application allowed users to search for items from an inventory file, add up to five items to a cart, calculate a running subtotal, apply quantity-based discounts, calculate tax, generate an invoice, and append completed transactions to a CSV log.

The interface required multiple input fields, shopping-cart display areas, and six main control buttons that changed state as the user moved through the purchase flow. The project emphasized GUI state management, file handling, event-driven programming, and business-logic simulation in Java.

Skills Demonstrated

  • Java GUI development
  • Event-driven programming
  • File I/O with CSV data
  • State-based interface logic
  • Invoice and transaction workflow design

Bike Database Interface

Bike Database Interface was a two-tier Java client-server application built with MySQL and JDBC. The application provided a GUI front end where users could connect to selected databases through properties files, enter SQL commands, execute queries and updates, and view results directly in the interface.

The project also included support for multiple client types with different permissions, a separate operations-log database, and an accountant-facing application for viewing transaction counts. It emphasized JDBC connectivity, GUI-based SQL execution, credential validation, database permissions, and client-server design.

Skills Demonstrated

  • Java Swing GUI development
  • JDBC and MySQL connectivity
  • Client-server application design
  • SQL command execution and results handling
  • Role-based database access concepts

GUI Board

GUI Board was a Java project centered on generating a randomized board made of points, rewards, obstacles, an entrance, and an exit. The board started as an empty grid, and the design option generated a new randomized board with outside borders, random interior obstacles, and rewards placed throughout the layout. Entrance and exit were randomly assigned to corner locations, while rewards and walls were placed so they did not overlap.

The project also included a menu system that allowed the user to design a new board, save the current board design, and load a previously saved board from file. The final implementation used a larger grid, randomized wall placement, multiple reward types, and persistent board saving/loading, which made the project a strong exercise in Java GUI logic, object placement, serialization, and interface design.

Skills Demonstrated

  • Java GUI development
  • Randomized board generation
  • Object placement using point-based coordinates
  • Save and load functionality through file handling
  • Menu-driven interface design
